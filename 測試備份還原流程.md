# 測試備份還原流程

## 📋 測試目的

驗證以下流程是否正常工作：
1. 新增使用者
2. 新增專案並設定預計開立發票月份
3. 手動備份
4. uninstall.sh（可選）
5. deploy.sh 重新部署（可選）
6. restore 還原
7. 驗證資料完整性

---

## 🧪 測試方法

### 方法一：使用自動測試腳本（推薦）⭐

```bash
# 1. 上傳測試腳本到伺服器
scp test-backup-restore.sh root@your-server:/opt/invoice-bonus-system/

# 2. SSH 登入伺服器
ssh root@your-server

# 3. 設定權限並執行測試
cd /opt/invoice-bonus-system
chmod +x test-backup-restore.sh
sudo ./test-backup-restore.sh
```

**測試腳本會自動：**
- ✅ 記錄當前資料（使用者、預計開票）
- ✅ 執行備份
- ✅ 模擬資料修改（刪除非admin使用者、清空預計開票）
- ✅ 執行還原
- ✅ 比對還原前後的資料
- ✅ 顯示詳細的差異報告

**預期輸出：**
```
============================================
  驗證結果
============================================

✓ 專案數量正確 (50 = 50)
✓ 使用者數量正確 (5 = 5)
✓ 預計開票數量正確 (15 = 15)
✓ 使用者帳號完全一致
✓ 預計開票資料完全一致

============================================
  ✅ 所有驗證通過！
============================================
```


## 🔍 常見問題排查

### 問題 1：還原後使用者帳號不見了

**可能原因：**
- 服務未正確重啟
- 資料庫檔案未正確替換

**排查方法：**
```bash
# 1. 檢查服務狀態
sudo systemctl status invoice-bonus-system.service

# 2. 檢查服務日誌
sudo journalctl -u invoice-bonus-system.service -n 50 --no-pager

# 3. 手動重啟服務
sudo systemctl restart invoice-bonus-system.service
sleep 5

# 4. 再次檢查資料
sqlite3 /opt/invoice-bonus-system/data/invoice_bonus.db "SELECT * FROM users;"
```

### 問題 2：還原後預計開票資料不見了

**可能原因：**
- 視圖未更新
- 資料庫欄位缺失

**排查方法：**
```bash
# 1. 檢查欄位是否存在
sqlite3 /opt/invoice-bonus-system/data/invoice_bonus.db "PRAGMA table_info(projects);" | grep expected

# 2. 檢查視圖定義
sqlite3 /opt/invoice-bonus-system/data/invoice_bonus.db "SELECT sql FROM sqlite_master WHERE type='view' AND name='v_project_summary';" | grep expected

# 3. 直接查詢 projects 表
sqlite3 /opt/invoice-bonus-system/data/invoice_bonus.db "SELECT project_code, expected_invoice_year_month FROM projects LIMIT 10;"

# 4. 如果欄位存在但視圖沒有，手動執行 restore.sh 的視圖更新部分
# （通常 restore.sh 應該已經自動處理）
```

### 問題 3：backup.sh 執行後沒有看到詳細統計

**可能原因：**
- 使用舊版 backup.sh
- sqlite3 不可用

**解決方法：**
```bash
# 1. 確認使用最新版本
cd /opt/invoice-bonus-system
ls -lh backup.sh

# 2. 檢查 sqlite3 是否可用
command -v sqlite3
# 如果沒有輸出，安裝 sqlite3
sudo apt-get install sqlite3

# 3. 重新執行備份
sudo ./backup.sh
```

### 問題 4：restore.sh 找不到備份檔案

**可能原因：**
- 備份目錄路徑錯誤
- 備份檔案被刪除

**排查方法：**
```bash
# 1. 檢查備份目錄
ls -lh /opt/invoice-bonus-backups/

# 2. 檢查備份檔案格式
ls -lh /opt/invoice-bonus-backups/backup_*.tar.gz

# 3. 如果找不到，檢查 uninstall 備份
ls -lh /opt/invoice-bonus-backups/uninstall_backup_*.tar.gz

# 4. 手動指定備份檔案
sudo ./restore.sh /opt/invoice-bonus-backups/backup_20251226_123456.tar.gz
```
