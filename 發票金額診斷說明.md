# 發票金額診斷說明

## 問題情境

- 曾直接修改發票號碼或發票日期
- 儀表板「已開立發票未收款總額」與專案管理「未收款」篩選金額有誤差

## 直接修改發票號碼/日期是否會造成重複？

**不會。** 若您是對同一筆發票做 `UPDATE`（修改 invoice_number、invoice_date），資料庫仍只有一筆發票，不會重複加總。

可能造成誤差的原因：

1. **儀表板與專案管理計算邏輯不同**
   - 儀表板：未排除軟刪除發票、未扣除部分折讓
   - 專案管理（v_project_summary）：排除軟刪除、扣除部分折讓

2. **疑似重複發票**
   - 同專案、同金額、不同發票號碼的兩筆以上發票

3. **作廢重開關聯**
   - 原發票與新發票的 `original_invoice_id` / `replacement_invoice_id` 設定不正確

## 執行診斷

### 方式一：sqlite3 命令列

```bash
cd /path/to/project
sqlite3 data/invoice_bonus.db < scripts/diagnose_invoice_amounts.sql
```

### 方式二：單一查詢（複製貼上）

在 sqlite3 或資料庫工具中執行：

```sql
-- 1. 三種計算方式總額比對
SELECT '儀表板邏輯(不含deleted_at、allowance)' as 方式, 
       COALESCE(SUM(amount_with_tax), 0) as 總額 
FROM invoices WHERE (status IS NULL OR status = '有效');
SELECT 'v_project_summary邏輯' as 方式, 
       COALESCE(SUM(amount_with_tax - COALESCE(allowance_amount, 0)), 0) as 總額 
FROM invoices WHERE (status IS NULL OR status = '有效') AND (deleted_at IS NULL);

-- 2. 有部分折讓或已軟刪除的有效發票
SELECT i.id, p.project_code, i.invoice_number, i.amount_with_tax, 
       i.allowance_amount, i.deleted_at
FROM invoices i JOIN projects p ON i.project_id = p.id
WHERE (i.status IS NULL OR i.status = '有效')
  AND (i.deleted_at IS NOT NULL OR COALESCE(i.allowance_amount, 0) > 0);

-- 3. 疑似重複（同專案同金額多筆）
SELECT project_id, amount_with_tax, COUNT(*) as 筆數,
       GROUP_CONCAT(invoice_number) as 發票號碼
FROM invoices 
WHERE (status IS NULL OR status = '有效') AND (deleted_at IS NULL)
GROUP BY project_id, amount_with_tax HAVING COUNT(*) > 1;
```

---

## 查詢結果解讀

| 查詢 | 若結果有資料 | 可能原因 |
|------|--------------|----------|
| 1. 三種總額比對 | 儀表板總額 > v_project_summary | 有軟刪除發票或部分折讓未扣除 |
| 2. 部分折讓/軟刪除 | 有資料 | 這些發票會造成儀表板與專案管理差異 |
| 3. 疑似重複 | 有資料 | 同專案同金額多筆發票，可能為重複開立 |
| 4. 專案金額比對 | 有差異 | 視圖與直接加總不一致，需檢查 |
| 5. 收款總額比對 | 含已刪除 > 僅未刪除 | 有軟刪除收款未排除 |
| 6. 作廢重開關聯 | 有資料 | 檢查 original/replacement 是否正確 |

---

## 修正建議

1. **若為軟刪除發票**：確認是否應還原或保持刪除；儀表板查詢應加入 `deleted_at IS NULL`。
2. **若為部分折讓**：儀表板查詢應改為 `amount_with_tax - allowance_amount`。
3. **若為重複發票**：確認是否應作廢其中一筆。
4. **若為作廢重開**：確認原發票 status=作廢、新發票 status=有效，且關聯正確。
