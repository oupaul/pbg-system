# 儀表板權限與角色編輯修復說明

## 問題描述

### 問題 1：儀表板權限控制不當
非管理員用戶也能看到以下敏感資訊：
- 通知系統狀態資訊（調試資訊）
- 總獎金統計
- 待發放獎金統計

**預期行為**：這些資訊應該只有管理員（admin）可以看到。

### 問題 2：角色編輯更新失敗
編輯已存在的角色後，點擊「更新角色」按鈕會一直顯示「處理中...」，無法完成更新。

## 解決方案

### 修復 1：儀表板權限控制

#### 修改檔案
`src/views/index.ejs`

#### 修改內容

**1. 通知系統狀態（第 34-48 行）**

修改前：
```javascript
${typeof showNotification !== 'undefined' && (!locals.user || locals.user.role !== 'boss') ? `
```

修改後：
```javascript
${typeof showNotification !== 'undefined' && locals.user && locals.user.role === 'admin' ? `
```

**說明**：只有管理員（admin）可以看到通知系統狀態資訊。

**2. 總獎金和待發放獎金卡片（第 160-189 行）**

修改前：
```javascript
${locals.user && locals.user.role === 'boss' ? '' : `
  <!-- 總獎金卡片 -->
  <!-- 待發放獎金卡片 -->
`}
```

修改後：
```javascript
${locals.user && locals.user.role === 'admin' ? `
  <!-- 總獎金卡片 -->
  <!-- 待發放獎金卡片 -->
` : ''}
```

**說明**：
- 原本邏輯：除了 boss 之外都顯示（包括 user, salesperson 等）
- 新邏輯：只有 admin 顯示，其他人（包括 user, salesperson, boss）都不顯示

## 常見問題

### Q1: 儀表板修改後，非 admin 使用者看不到任何統計資訊？

**A**: 這是預期行為（針對 salesperson 和 boss）。但 user 角色應該能看到：
- 總專案數
- 總金額(含稅)
- 已開立發票總額
- 未開立發票金額
- 專案類型分布

如果 user 也看不到，請檢查 `src/views/index.ejs` 第 2 行：
```javascript
${locals.user && locals.user.role === 'salesperson' ? `
```

確保 user 角色不在此判斷內。

### Q2: 角色編輯時按鈕一直顯示「處理中」？

**A**: 檢查步驟：

1. **檢查瀏覽器 Console**：
   - 按 F12 開啟開發者工具
   - 切換到 Console 面板
   - 查看是否有錯誤訊息

2. **檢查網絡請求**：
   - 在開發者工具切換到 Network 面板
   - 重新提交表單
   - 查看 PUT /roles/:id 請求：
     - Status 是否為 200
     - Response 是否為 `{success: true, ...}`

3. **檢查服務器日誌**：
   ```bash
   sudo journalctl -u project-system.service -n 100 --no-pager | grep "更新角色"
   ```

4. **常見原因**：
   - 服務未重啟（`sudo systemctl restart project-system.service`）
   - 檔案上傳不完整（重新上傳檔案）
   - 資料庫鎖定（重啟服務）
   - 網絡問題（檢查防火牆）

### Q3: 更新後部分權限沒有正確儲存？

**A**: 這可能是 checkbox 處理問題。驗證步驟：

1. 在瀏覽器 Console 查看「表單資料」日誌
2. 確認未勾選的 checkbox 不在資料中（這是正確的）
3. 檢查後端日誌中的「更新資料」
4. 確認 `can_edit: 0` 等欄位正確（0 表示未勾選）

## 技術細節

### 權限控制邏輯

**原邏輯（有問題）**：
```javascript
// 通知系統狀態
!locals.user || locals.user.role !== 'boss'
// 意思：未登入 OR 角色不是 boss → 顯示
// 結果：admin, user, salesperson 都會顯示

// 獎金統計
locals.user.role === 'boss' ? '' : '顯示'
// 意思：角色是 boss → 隱藏，否則顯示
// 結果：admin, user, salesperson 都會顯示
```

**新邏輯（正確）**：
```javascript
// 通知系統狀態和獎金統計
locals.user && locals.user.role === 'admin'
// 意思：已登入 AND 角色是 admin → 顯示
// 結果：只有 admin 顯示，其他人都隱藏
```

### 角色編輯表單提交流程

```
1. 使用者點擊「更新角色」
   ↓
2. handleSubmit() 阻止表單默認提交
   ↓
3. 收集表單資料（FormData）
   ↓
4. 使用 fetch API 發送 PUT 請求
   - URL: /roles/:id
   - Method: PUT
   - Content-Type: application/x-www-form-urlencoded
   - Body: URLSearchParams(formData)
   ↓
5. 後端 Express 接收請求
   - 解析 URL 參數（:id）
   - 解析 Body（express.urlencoded）
   ↓
6. 後端驗證和處理
   - 驗證必填欄位
   - 檢查角色存在
   - 處理 checkbox（'on' → 1, undefined → 0）
   - 調用 Role.update()
   ↓
7. 返回 JSON 響應
   - 成功：{success: true, message: "角色更新成功"}
   - 失敗：{error: "錯誤訊息"}
   ↓
8. 前端處理響應
   - 成功：跳轉到 /roles?success=...
   - 失敗：顯示錯誤訊息，恢復按鈕狀態
```

### Checkbox 處理

**問題**：未勾選的 checkbox 不會出現在 FormData 中

**解決方案**：後端使用三元運算符處理
```javascript
can_edit: req.body.can_edit === 'on' ? 1 : 0
```

**邏輯**：
- 勾選：`req.body.can_edit === 'on'` → 1
- 未勾選：`req.body.can_edit === undefined` → 0
