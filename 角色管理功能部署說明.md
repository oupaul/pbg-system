# 角色管理功能部署說明

## 功能簡介

新增的角色管理功能允許管理員：
- 查看和管理系統中的所有角色
- 創建自訂角色
- 為每個角色設定細緻的權限
- 查看每個角色的使用者數量
- 系統預設提供 4 個基本角色

## 系統預設角色

| 角色代碼 | 角色名稱 | 權限說明 |
|---------|---------|---------|
| `admin` | 系統管理員 | 擁有所有權限，可管理使用者、角色、系統設定等 |
| `user` | 一般使用者 | 可以編輯和刪除資料，但不能管理系統設定 |
| `salesperson` | 業務員 | 只能查看自己負責的專案，無法編輯資料 |
| `boss` | 老闆 | 可查看所有專案和報表，但無法編輯資料 |

## 權限類型

### 1. 資料操作權限
- **編輯資料**：可以新增、修改專案、發票、收款等資料
- **刪除資料**：可以刪除專案、發票、收款等資料

### 2. 系統管理權限
- **使用者管理**：可以管理系統使用者
- **角色管理**：可以管理角色和權限
- **系統設定**：可以修改系統設定
- **備份還原**：可以執行備份和還原操作

### 3. 專案檢視權限
- **檢視所有專案**：可以查看系統中的所有專案
- **檢視自己的專案**：只能查看自己負責的專案（需綁定業務員）

## 部署步驟

### 方法一：使用一鍵部署腳本（推薦）

```bash
cd /opt/invoice-bonus-system
sudo ./deploy.sh
```

部署腳本會自動執行角色管理資料表遷移。

### 方法二：手動部署

如果需要手動部署，請依序執行以下步驟：

```bash
# 1. 停止服務
sudo systemctl stop invoice-bonus-system.service

# 2. 進入專案目錄
cd /opt/invoice-bonus-system

# 3. 備份資料庫
cp data/invoice_bonus.db data/invoice_bonus.db.backup-$(date +%Y%m%d_%H%M%S)

# 4. 執行角色管理資料表遷移
npm run migrate:roles

# 5. 重啟服務
sudo systemctl start invoice-bonus-system.service

# 6. 檢查服務狀態
sudo systemctl status invoice-bonus-system.service
```

## 驗證部署

### 1. 檢查資料表是否建立成功

```bash
cd /opt/invoice-bonus-system
sqlite3 data/invoice_bonus.db "SELECT name FROM sqlite_master WHERE type='table' AND name='roles';"
```

應該看到：
```
roles
```

### 2. 檢查預設角色是否插入

```bash
sqlite3 data/invoice_bonus.db "SELECT role_key, role_name FROM roles ORDER BY display_order;"
```

應該看到：
```
admin|系統管理員
user|一般使用者
salesperson|業務員
boss|老闆
```

### 3. 登入系統測試

1. 使用管理員帳號登入系統
2. 在導航選單中應該會看到「角色管理」選項
3. 點擊進入角色管理頁面
4. 應該會看到 4 個預設角色

## 使用方式

### 查看角色列表

1. 登入系統（需要管理員權限）
2. 點擊導航選單的「角色管理」
3. 查看所有角色及其權限設定

### 新增自訂角色

1. 進入角色管理頁面
2. 點擊「新增角色」按鈕
3. 填寫角色資訊：
   - 角色代碼（英文，用於系統識別）
   - 角色名稱（中文，顯示名稱）
   - 描述（選填）
   - 顯示順序
4. 勾選權限設定
5. 點擊「建立角色」

### 編輯角色

1. 在角色列表中找到要編輯的角色
2. 點擊「編輯」按鈕
3. 修改角色資訊或權限設定
4. 點擊「更新角色」

**注意**：系統預設角色的 `role_key` 不能修改。

### 刪除角色

1. 在角色列表中找到要刪除的角色
2. 點擊「刪除」按鈕
3. 確認刪除

**注意**：
- 系統預設角色不能刪除
- 如果有使用者正在使用該角色，無法刪除
- 刪除操作是軟刪除（停用），不會永久刪除資料

### 查看角色詳情

1. 在角色列表中點擊角色名稱或「檢視」按鈕
2. 可以看到：
   - 角色的完整資訊
   - 所有權限設定
   - 使用此角色的使用者列表

## 資料庫結構

### roles 表

| 欄位名稱 | 類型 | 說明 |
|---------|------|------|
| id | INTEGER | 主鍵 |
| role_key | TEXT | 角色代碼（唯一） |
| role_name | TEXT | 角色名稱 |
| description | TEXT | 描述 |
| can_edit | INTEGER | 可編輯資料 (0/1) |
| can_delete | INTEGER | 可刪除資料 (0/1) |
| can_manage_users | INTEGER | 可管理使用者 (0/1) |
| can_manage_roles | INTEGER | 可管理角色 (0/1) |
| can_manage_settings | INTEGER | 可管理系統設定 (0/1) |
| can_backup_restore | INTEGER | 可備份還原 (0/1) |
| can_view_all_projects | INTEGER | 可檢視所有專案 (0/1) |
| can_view_own_projects | INTEGER | 可檢視自己的專案 (0/1) |
| is_system_role | INTEGER | 是否為系統角色 (0/1) |
| is_active | INTEGER | 是否啟用 (0/1) |
| display_order | INTEGER | 顯示順序 |
| created_at | TEXT | 創建時間 |
| updated_at | TEXT | 更新時間 |

## 注意事項

1. **權限變更立即生效**：修改角色權限後，使用該角色的所有使用者權限會立即更新（需要重新登入）

2. **系統角色保護**：系統預設的 4 個角色標記為系統角色，不能刪除，`role_key` 不能修改

3. **使用者關聯檢查**：刪除角色前會檢查是否有使用者正在使用，確保資料一致性

4. **審計日誌**：所有角色的建立、修改、刪除操作都會記錄在審計日誌中

5. **備份建議**：在大量修改角色權限前，建議先執行備份

## 常見問題

### Q: 如何為現有使用者分配新角色？

A: 進入「使用者管理」頁面，編輯使用者資料，從角色下拉選單中選擇新角色。

### Q: 可以創建多個具有相同權限的角色嗎？

A: 可以。角色代碼（role_key）必須唯一，但權限設定可以相同。

### Q: 修改角色權限後，使用者需要重新登入嗎？

A: 建議重新登入以確保權限立即生效。系統會在下次登入時載入最新的角色權限。

### Q: 如何停用某個角色？

A: 編輯角色時，取消勾選「啟用此角色」，儲存後該角色就會被停用。

### Q: 停用角色後，使用該角色的使用者會怎樣？

A: 使用停用角色的使用者將無法登入系統。建議先將使用者轉移到其他角色再停用。
