# 閒置自動登出功能說明

## 📋 功能概述

為提升系統安全性，系統新增了「使用者閒置自動登出」功能。當使用者在設定的時間內沒有任何操作時，系統會自動登出以防止未經授權的存取。

## 🎯 功能特色

### 1. 智能閒置檢測
- **活動監聽**：自動偵測使用者活動（滑鼠移動、鍵盤輸入、點擊、滾動、觸控等）
- **自動重置**：任何使用者活動都會自動重置閒置計時器
- **節流優化**：使用節流機制避免過度計算，提升效能

### 2. 友善的警告機制
- **提前警告**：在達到閒置時間前會顯示警告對話框
- **倒數計時**：清楚顯示剩餘時間（秒數）
- **兩種選擇**：
  - 「繼續使用」：延長會話時間
  - 「立即登出」：主動登出

### 3. 可配置的設定
- **閒置時間**：0-480 分鐘（0-8 小時），0 表示停用此功能
- **警告時間**：1-10 分鐘，在閒置時間前多久顯示警告
- **即時生效**：設定變更後重新整理頁面即可生效

### 4. 安全性保障
- **自動登出**：如果警告期間仍無操作，系統會自動登出
- **清除會話**：登出後會完全清除伺服器端的會話資料
- **返回登入頁**：自動重定向到登入頁面

## ⚙️ 系統設定

### 存取設定頁面

1. 使用**管理員帳號**登入系統
2. 點擊左側選單「系統設定」
3. 找到「閒置自動登出設定」區塊

### 設定項目說明

#### 閒置時間（分鐘）
- **描述**：使用者在此時間內沒有任何操作將被自動登出
- **範圍**：0-480 分鐘（0-8 小時）
- **預設值**：30 分鐘
- **建議值**：
  - **30 分鐘**：標準設定（推薦）
  - **60 分鐘**：較寬鬆的時間限制
  - **15 分鐘**：更嚴格的安全性
  - **0 分鐘**：停用閒置登出功能

#### 警告提前時間（分鐘）
- **描述**：在達到閒置時間前多久顯示警告對話框
- **範圍**：1-10 分鐘
- **預設值**：2 分鐘
- **說明**：在自動登出前 2 分鐘會顯示警告對話框，讓使用者有機會延長會話時間

## 🚀 部署與更新

### 首次部署（已包含此功能）

執行標準部署流程即可：


部署腳本會自動：
1. 執行 `migrate:system-settings` 遷移
2. 創建閒置登出的設定項（如果不存在）
3. 設定預設值（閒置時間：30 分鐘，警告時間：2 分鐘）

### 從舊版本更新

如果您是從舊版本更新，執行部署腳本即可：

```bash

scp -r ./* username@your-server:/path/to/project/

# 2. 登入伺服器並執行部署
ssh username@your-server
cd /path/to/project
sudo ./deploy.sh
```

系統會自動：
- 更新資料庫結構
- 添加新的設定項
- 保留現有的其他設定


## 💡 使用說明

### 一般使用者

1. **正常使用**：正常操作系統時，閒置計時器會自動重置
2. **看到警告**：如果看到警告對話框，表示您即將被登出
   - 點擊「繼續使用」延長會話時間
   - 點擊「立即登出」主動登出
3. **自動登出**：如果警告期間仍無操作，系統會自動登出並跳轉到登入頁面

### 管理員

1. **檢查設定**：進入「系統設定」頁面查看當前配置
2. **調整時間**：根據實際需求調整閒置時間和警告時間
3. **停用功能**：如果不需要此功能，將「閒置時間」設定為 0
4. **通知使用者**：設定變更後，建議通知所有使用者重新整理頁面

## 🔍 功能驗證

### 測試步驟

1. **登入系統**
2. **保持閒置**：不進行任何操作
3. **等待警告**：在設定的時間前 2 分鐘，應該會看到警告對話框
4. **測試延長**：點擊「繼續使用」，計時器應該重置
5. **測試自動登出**：保持閒置直到倒數結束，應該會自動登出

### 檢查配置

使用瀏覽器開發者工具（F12）的控制台，應該會看到：

```
[閒置檢測] 已啟用
[閒置檢測] 閒置時間: 30 分鐘
[閒置檢測] 警告時間: 2 分鐘
[閒置檢測] 自動登出時間: 30 分鐘後
```

如果顯示：
```
[閒置檢測] 已停用（閒置時間設定為 0）
```
表示此功能已被停用。

## 🔧 技術實現

### 前端實現 (public/js/main.js)

- 使用 JavaScript `setTimeout` 實現閒置計時器
- 監聽多種使用者活動事件（mousedown, mousemove, keypress, scroll, touchstart, click）
- 使用節流機制避免過度觸發（每秒最多一次）
- Bootstrap Modal 顯示警告對話框
- 倒數計時使用 `setInterval` 每秒更新

### 後端實現

#### 資料庫遷移 (migrations/migrate_system_settings.js)
- 創建 system_settings 表
- 插入預設的閒置登出設定

#### 全域中介軟體 (src/app.js)
- 從資料庫讀取閒置登出設定
- 透過 `res.locals` 傳遞給所有視圖

#### 視圖模板 (src/views/layout.ejs)
- 將伺服器設定嵌入到 JavaScript 全域變數 `window.idleConfig`
- 前端腳本從此變數讀取配置

#### 系統設定頁面 (src/views/settings/index.ejs)
- 提供管理員介面調整閒置登出設定
- 包含輸入驗證和建議值說明

#### 設定驗證 (src/routes/settings.js)
- 閒置時間：0-480 分鐘
- 警告時間：1-10 分鐘

## 🎨 使用者體驗

### 警告對話框設計

- **標題**：黃色背景，警告圖示
- **內容**：清楚說明剩餘時間和原因
- **按鈕**：
  - 主要按鈕（藍色）：繼續使用
  - 次要按鈕（灰色）：立即登出
- **倒數計時**：粗體顯示剩餘秒數
- **無法關閉**：使用 `data-bs-backdrop="static"` 防止點擊外部關閉

### Toast 提示

- **延長會話**：顯示綠色成功提示「會話時間已延長」
- **自動登出**：顯示黃色警告提示「由於長時間未操作，您已被自動登出」

## 🛡️ 安全性考量

### 保護機制

1. **會話清除**：登出時完全銷毀伺服器端 session
2. **強制重定向**：自動登出後立即跳轉到登入頁面
3. **無法繞過**：前端檢測配合後端 session 管理

### 注意事項

- **多分頁**：每個分頁獨立計時，不同分頁可能有不同的登出時間
- **瀏覽器限制**：閒置計時器在分頁切換或瀏覽器最小化時仍然運作
- **網路斷線**：如果網路斷線，前端仍會顯示警告並嘗試登出
- **共用電腦**：此功能特別適合在共用電腦環境使用

## 📝 常見問題

### Q: 設定變更後何時生效？

A: 設定變更後，需要使用者重新整理頁面才會生效。建議在非尖峰時段修改設定，並通知所有使用者重新整理瀏覽器。

### Q: 可以完全停用此功能嗎？

A: 可以，將「閒置時間」設定為 0 即可停用閒置登出功能。

### Q: 如果使用者正在輸入資料會被登出嗎？

A: 不會，任何鍵盤輸入都會重置閒置計時器。只有在完全無操作的情況下才會觸發警告。

### Q: 警告時間可以設定為 0 嗎？

A: 不行，警告時間最少需要 1 分鐘，以確保使用者有足夠時間看到警告並做出反應。

### Q: 多個分頁都會顯示警告嗎？

A: 是的，每個分頁都會獨立顯示警告。點擊任一分頁的「繼續使用」只會延長該分頁的會話。

### Q: 登出時會儲存未完成的資料嗎？

A: 不會，自動登出不會自動儲存任何資料。建議定期儲存重要資料。
