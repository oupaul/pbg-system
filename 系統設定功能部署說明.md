# 系統設定功能部署說明

## 📋 功能概述

現在管理者可以在網頁介面直接調整開票提醒通知的提前天數，無需修改代碼和重啟服務。

**功能包括：**
- ✅ 可設定通知提前天數（0-30天）
- ✅ 可啟用/停用通知功能
- ✅ 設定即時生效，無需重啟服務
- ✅ 所有設定變更都會記錄在審計日誌中

---

## 🚀 部署步驟

### 1️⃣ 上傳檔案

將以下檔案上傳到伺服器：

```bash
# 必須上傳的檔案
src/routes/settings.js              # 系統設定路由
src/views/settings/index.ejs        # 系統設定頁面
src/routes/index.js                 # 修改：從資料庫讀取設定
src/app.js                          # 修改：註冊設定路由
src/views/layout.ejs                # 修改：添加導航連結
migrations/migrate_system_settings.js  # 資料庫遷移檔案
```

### 2️⃣ 執行資料庫遷移

SSH 登入伺服器並執行遷移：

```bash
ssh root@pbg-dev.m7r.tw

cd /opt/invoice-bonus-system

# 執行遷移（建立資料表和插入預設值）
node migrations/migrate_system_settings.js
```

**預期輸出：**
```
開始執行系統設定資料表遷移...
✓ 建立 system_settings 表
✓ 插入設定：invoice_notification_days_before_month_end = 6
✓ 插入設定：invoice_notification_enabled = true
✓ 系統設定資料表遷移完成
```

### 3️⃣ 重啟服務

```bash
sudo systemctl restart invoice-bonus-system

# 驗證服務狀態
sudo systemctl status invoice-bonus-system
```

## 🎯 使用說明

### 進入系統設定頁面

1. **登入系統**（必須是 admin 角色）
2. 點擊導航欄的 **「系統設定」** 連結（齒輪圖示）
3. 或直接訪問：`https://pbg-dev.m7r.tw/settings`

### 調整通知提前天數

1. 在「通知提前天數」欄位輸入數值（0-30）
2. 點擊「儲存設定」
3. **設定立即生效**，無需重啟服務

**建議值：**
- **6 天**：倒數第 7 天開始（推薦，預設值）
- **9 天**：倒數第 10 天開始（提前更多）
- **2 天**：倒數第 3 天開始（緊急提醒）

### 啟用/停用通知

1. 切換「通知功能狀態」開關
2. 點擊「儲存設定」
3. 如果停用，儀表板將不再顯示通知

## 🔍 驗證功能

### 測試步驟

1. **進入系統設定頁面**
   - 確認可以看到「開票提醒通知設定」區塊

2. **修改通知天數**
   - 將提前天數改為 `9`
   - 點擊「儲存設定」
   - 看到成功訊息

3. **驗證通知顯示**
   - 返回儀表板（首頁）
   - 如果條件滿足（剩餘天數 ≤ 9），應該會看到通知

4. **測試停用功能**
   - 回到系統設定頁面
   - 取消勾選「通知功能狀態」
   - 點擊「儲存設定」
   - 返回儀表板，通知應該消失

## 📊 資料庫結構

### system_settings 表

| 欄位 | 類型 | 說明 |
|------|------|------|
| id | INTEGER | 主鍵 |
| setting_key | TEXT | 設定鍵值（唯一） |
| setting_value | TEXT | 設定值 |
| setting_type | TEXT | 類型（string/number/boolean/json） |
| description | TEXT | 說明 |
| created_at | TEXT | 建立時間 |
| updated_at | TEXT | 更新時間 |

### 預設設定

| setting_key | setting_value | setting_type | 說明 |
|-------------|---------------|--------------|------|
| invoice_notification_days_before_month_end | 6 | number | 通知提前天數（預設6天） |
| invoice_notification_enabled | true | boolean | 通知功能啟用狀態（預設啟用） |

## 🔧 技術細節

### 設定讀取邏輯

**首頁（儀表板）自動從資料庫讀取設定：**

```javascript
// src/routes/index.js
// 從系統設定讀取通知天數（預設6天）
const notificationDaysSetting = db.prepare(
  'SELECT setting_value FROM system_settings WHERE setting_key = ?'
).get('invoice_notification_days_before_month_end');

const notificationDaysBeforeMonth = notificationDaysSetting 
  ? parseInt(notificationDaysSetting.setting_value, 10) 
  : 6; // 預設值
```

### 設定更新邏輯

**設定路由處理更新：**

```javascript
// src/routes/settings.js
// 更新設定並記錄審計日誌
const success = updateSystemSetting(key, value, type);
if (success) {
  AuditLogService.logUpdate('system_settings', key, oldValue, newValue, user);
}
```

## ❓ 常見問題

### Q1: 設定後通知沒有顯示？

**檢查清單：**
- [ ] 通知功能是否已啟用？
- [ ] 當前日期是否在通知期內（剩餘天數 ≤ 設定值）？
- [ ] 是否有符合條件的專案（預計開票年月 = 當前月份，狀態 = 未結案）？
- [ ] 是否重新整理了頁面？

### Q2: 如何查看設定變更記錄？

所有設定變更都會記錄在「修改記錄」頁面：
1. 進入「修改記錄」
2. 篩選「資料表」為 `system_settings`
3. 查看所有設定變更歷史

### Q3: 可以添加其他系統設定嗎？

**可以！** 只需要：

1. **在資料庫添加設定：**
```bash
sqlite3 /opt/invoice-bonus-system/data/invoice_bonus.db
INSERT INTO system_settings (setting_key, setting_value, setting_type, description)
VALUES ('your_setting_key', 'default_value', 'string', '設定說明');
```

2. **在設定頁面添加輸入欄位：**
編輯 `src/views/settings/index.ejs`，添加新的表單欄位

3. **在代碼中使用設定：**
```javascript
const setting = db.prepare('SELECT setting_value FROM system_settings WHERE setting_key = ?')
  .get('your_setting_key');
```
