# 角色功能完整修復總結

## 修復概述

本次修復解決了角色管理功能上線後的兩個關鍵問題：

1. ✅ **使用者管理角色選單硬編碼** → 無法選取自訂角色
2. ✅ **資料庫 CHECK 約束限制** → 分配自訂角色時報錯

## 問題 1：使用者管理角色選單硬編碼

### 問題描述
在角色管理中新增自訂角色後，這些角色無法在使用者管理的角色選單中顯示。

### 原因
使用者管理頁面的角色選單是硬編碼的，只顯示 4 個預設角色。

### 解決方案

修改了 3 個檔案：

#### 1. `src/routes/users.js`
- 在 3 處路由中新增動態讀取角色列表的邏輯
- 使用者列表：查詢所有角色名稱（`rolesMap`）
- 新增/編輯表單：查詢所有啟用的角色（`roles`）

#### 2. `src/views/users/form.ejs`
- 角色選單改為動態生成
- 從 `roles` 陣列動態生成 `<option>` 元素
- 保留向後兼容（如果 `roles` 不存在，顯示預設角色）

#### 3. `src/views/users/index.ejs`
- 使用者列表動態顯示角色名稱
- 從 `rolesMap` 讀取角色顯示名稱
- 自訂角色使用深灰色徽章（`bg-dark`）

## 問題 2：資料庫 CHECK 約束限制

### 問題描述
分配自訂角色給使用者時，出現錯誤：
```
更新使用者失敗：CHECK constraint failed: role IN ('admin', 'user', 'salesperson', 'boss')
```

### 原因
`users` 表的 `role` 欄位有 CHECK 約束，限制只能使用 4 個預設角色值。

### 解決方案

創建了遷移腳本移除 CHECK 約束：

#### 新增檔案

1. **`migrations/migrate_remove_user_role_check.js`**
   - 檢測並移除 `users.role` 的 CHECK 約束
   - 使用事務確保資料安全
   - 失敗時自動回滾

2. **`fix-user-role-constraint.sh`**
   - 一鍵修復腳本
   - 自動備份、遷移、驗證
   - 適合快速修復現有環境

3. **`角色CHECK約束修復指南.md`**
   - 詳細的修復指南
   - 3 種修復方法（自動腳本、手動遷移、SQL 直接修復）
   - 常見問題解答

#### 修改檔案

1. **`package.json`**
   - 新增命令：`"migrate:remove-user-role-check"`

2. **`deploy.sh`**
   - 在兩處遷移流程中加入新遷移
   - 執行順序：`migrate:roles` → `migrate:remove-user-role-check` → `migrate:system-settings`

### 相關文件
- [角色CHECK約束修復指南.md](角色CHECK約束修復指南.md)

## 技術實現

### 角色選單動態化

**查詢邏輯**：
```javascript
// 查詢所有啟用的角色
const roles = db.prepare(`
  SELECT role_key, role_name 
  FROM roles 
  WHERE is_active = 1 
  ORDER BY display_order ASC, role_name ASC
`).all();
```

**視圖渲染**：
```javascript
// 動態生成選項
${roles.map(r => 
  '<option value="' + r.role_key + '"' + 
  (user && user.role === r.role_key ? ' selected' : '') + 
  '>' + r.role_name + '</option>'
).join('')}
```

### CHECK 約束移除

**遷移策略**：
1. 創建新表（沒有 CHECK 約束）
2. 複製所有資料
3. 刪除舊表
4. 重新命名新表
5. 重建索引

**為什麼不直接修改表**：
SQLite 不支援直接修改 CHECK 約束，必須重建表。

## 常見問題

### Q1: 更新後還是無法使用自訂角色？

**檢查清單**：
```bash
# 1. 確認遷移已執行
cd /opt/project-system
npm run migrate:remove-user-role-check

# 2. 確認服務已重啟
sudo systemctl restart project-system.service

# 3. 檢查資料庫結構
sqlite3 data/invoice_bonus.db "SELECT sql FROM sqlite_master WHERE type='table' AND name='users';"

# 4. 清除瀏覽器快取
```

### Q2: 遷移時報錯怎麼辦？

**處理步驟**：
```bash
# 1. 查看詳細錯誤訊息
npm run migrate:remove-user-role-check

# 2. 檢查資料庫是否被鎖定
sudo systemctl stop project-system.service

# 3. 重試遷移
npm run migrate:remove-user-role-check

# 4. 如果還是失敗，使用快速修復腳本
cd /root
sudo ./fix-user-role-constraint.sh
```

### Q3: 如何回退到舊版本？

**不建議回退**，但如果必須：
```bash
# 1. 停止服務
sudo systemctl stop project-system.service

# 2. 還原資料庫備份
cd /opt/project-system
cp data/invoice_bonus.db.backup-YYYYMMDD_HHMMSS data/invoice_bonus.db

# 3. 還原舊版本代碼
# （從備份或 Git 恢復）

# 4. 重啟服務
sudo systemctl start project-system.service
```

**注意**：回退後自訂角色的使用者將無法正常使用。

### Q4: 自訂角色的徽章顏色可以改嗎？

可以，需要進一步開發：
1. 在 `roles` 表新增 `badge_color` 欄位
2. 在角色管理表單新增顏色選擇器
3. 修改 `users/index.ejs` 讀取該欄位

預設情況下，自訂角色使用深灰色（`bg-dark`）。

### Q5: 能否限制某些使用者只能分配特定角色？

目前不支援，所有管理員可以分配任何啟用的角色。如需此功能，需要開發角色權限細化功能。
