# 一筆發票分多次收款功能說明

## 更新日期
2026-02-12（含收款狀態修正）

---

## 一、功能概述

支援同一張發票可對應多筆收款記錄，適用於客戶分批付款的情境。系統會自動彙總該發票的所有收款，正確計算已收款與未收款金額。

---

## 二、功能說明

### 2.1 新增收款

- **發票選擇**：下拉選單顯示專案內所有有效發票，**可重複選擇同一發票**
- **顯示格式**：`發票號碼 (總額 $X，未收 $Y)` 或 `發票號碼 (總額 $X，已收齊)`
- **說明文字**：「支援一筆發票分多次收款，可重複選擇同一發票」
- **篩選邏輯**：僅顯示 `isValid !== false` 的發票（排除已作廢、整筆折讓）

### 2.2 編輯收款

- **發票選擇**：下拉選單顯示所有發票（含已作廢、折讓），確保編輯時可正確顯示目前選中的發票
- **已作廢/折讓標示**：非有效發票在選項後加 `(已作廢/折讓)`
- **顯示格式**：與新增收款相同，顯示總額與未收/已收齊狀態

### 2.3 計算邏輯

- **發票認列金額**：`amount_with_tax - allowance_amount`（扣除部分折讓）
- **已收款**：該發票所有收款記錄的 `calculateActualReceived` 加總（含匯費差異）
- **未收款**：`max(0, 認列金額 - 已收款)`

### 2.4 發票明細收款狀態

發票明細表格中的「收款狀態」欄位，依是否已收齊區分：

**部分收款（尚未收齊）**：不顯示「提前到款」，改依預計收款日顯示待收狀態

| 條件 | 顯示 |
|------|------|
| 無預計收款日 | 部分收款 (未收 $X) |
| 預計收款日 > 今天 | 部分收款，將於 n 天後收款 |
| 預計收款日 < 今天 | 部分收款，已逾期 n 天 |
| 預計收款日 = 今天 | 部分收款，預計今日收款 |

**已收齊**：以最後一筆收款日期與預計收款日比較

| 條件 | 顯示 |
|------|------|
| 最後收款日 = 預計日 | 準時到款 |
| 最後收款日 > 預計日 | 款項到帳但逾期 n 天 |
| 最後收款日 < 預計日 | 提前 n 天到款 |

**無收款記錄**：依預計收款日顯示「將於 n 天後收款」或「已逾期 n 天」

> 修正原因：分次收款時預計收款日會持續更新為下次應收日，若以第一筆收款日期比較會誤顯示「提前到款」。改為僅在已收齊時才顯示提前/準時/逾期到款。

## 三、技術實作

### 3.1 後端：invoiceUnpaidSummary

**位置**：`src/routes/projects.js`

**計算流程**：
1. 以 `validInvoices` 為基礎
2. 加入「有收款記錄但不在 validInvoices」的發票（如已作廢但曾有收款）
3. 對每張發票計算：`amount`、`paid`、`unpaid`、`isValid`

**資料結構**：
```javascript
{
  invoice_id: number,
  invoice_number: string,
  amount: number,    // 認列金額
  paid: number,      // 已收款
  unpaid: number,    // 未收款
  isValid: boolean   // 是否有效（非作廢/折讓）
}
```

### 3.2 前端：新增/編輯收款 Modal

**位置**：`src/views/projects/show.ejs`

- **新增收款**：使用 `invoiceUnpaidSummary.filter(s => s.isValid !== false)`
- **編輯收款**：使用完整 `invoiceUnpaidSummary`，已作廢發票加 `(已作廢/折讓)` 後綴

### 3.3 既有後端邏輯（無需修改）

- `ReceivablesAgingService`：依 `invoice_id` 彙總多筆收款
- `Payment.calculateActualReceived`：計算單筆收款實際金額（含匯費差異）
- `v_project_summary` 視圖：已正確處理多筆收款

## 六、使用情境範例

**情境**：發票 AB-12345678 金額 $100,000，客戶分三次付款

1. 第一次收款 $30,000 → 新增收款，選擇發票 AB-12345678
2. 第二次收款 $40,000 → 新增收款，再次選擇發票 AB-12345678
3. 第三次收款 $30,000 → 新增收款，再次選擇發票 AB-12345678

**顯示**：
- 新增第二筆時：下拉顯示 `AB-12345678 (總額 $100,000，未收 $70,000)`
- 新增第三筆時：下拉顯示 `AB-12345678 (總額 $100,000，未收 $30,000)`
- 收齊後：下拉顯示 `AB-12345678 (總額 $100,000，已收齊)`
