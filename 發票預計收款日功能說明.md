# 發票預計收款日功能說明

## 更新日期
2026-02-02

## 功能概述

在專案管理的發票明細中，每筆發票可記錄「預計收款日」，便於追蹤應收帳款與收款時程。

## 功能特點

- **選填欄位**：預計收款日為選填，不強制輸入
- **新增發票**：新增發票時可一併設定預計收款日
- **編輯發票**：可隨時修改或清空預計收款日
- **列表顯示**：發票明細表格顯示預計收款日，未設定時顯示 `-`

## 資料庫

### 欄位說明
- **表名**：`invoices`
- **欄位**：`expected_payment_date` (TEXT, NULL)
- **格式**：YYYY-MM-DD（與 HTML date input 一致）

### 遷移
- **腳本**：`migrations/migrate_invoice_expected_payment_date.js`
- **指令**：`npm run migrate:invoice-expected-payment-date`
- **自動執行**：應用啟動時會自動檢查並添加欄位（若不存在）
- **部署**：deploy.sh 已整合此遷移，部署時會自動執行

## 向後兼容

- 既有發票的 `expected_payment_date` 預設為 NULL
- 應用啟動時會自動檢查 `invoices` 表是否有此欄位
- 若不存在則自動執行 ALTER TABLE 添加，無需手動執行遷移
- 不影響既有發票與收款功能
