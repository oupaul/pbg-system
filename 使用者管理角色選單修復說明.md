# 使用者管理角色選單修復說明

## 問題描述

在角色管理功能上線後，使用者管理頁面的角色選單仍然是硬編碼的，導致：
- ❌ 手動新增的角色無法在使用者管理中選取
- ❌ 只能選擇 4 個預設角色（admin, user, salesperson, boss）
- ❌ 使用者列表只顯示預設角色名稱

## 修復內容

### 1. 修改檔案清單

| 檔案 | 修改內容 |
|------|---------|
| `src/routes/users.js` | 新增動態讀取角色列表的邏輯（3處） |

| `src/views/users/index.ejs` | 使用者列表動態顯示角色名稱 |

**修改 3：編輯使用者表單（第96-114行）**
- 與修改 2 相同，新增角色列表查詢邏輯

#### src/views/users/form.ejs

**修改：角色選單（第47-56行）**

原本：
```html
<select name="role" class="form-select" id="roleSelect" required>
  <option value="user">一般使用者（可編輯）</option>
  <option value="admin">管理員（可編輯）</option>
  <option value="salesperson">業務員（唯讀）</option>
  <option value="boss">老闆（唯讀）</option>
</select>
```

修改後：
```html
<select name="role" class="form-select" id="roleSelect" required>
  ${typeof roles !== 'undefined' && roles.length > 0 ? 
    roles.map(r => '<option value="' + r.role_key + '"' + 
                    (user && user.role === r.role_key ? ' selected' : '') + 
                    '>' + r.role_name + '</option>').join('') 
    : 
    '<!-- 預設角色 -->'
  }
</select>
```

#### src/views/users/index.ejs

**修改：角色顯示（第52-57行）**

原本：
```javascript
${user.role === 'admin' ? '<span class="badge bg-danger">管理員</span>' : 
  user.role === 'boss' ? '<span class="badge bg-primary">老闆</span>' :
  user.role === 'salesperson' ? '<span class="badge bg-info">業務員</span>' :
  '<span class="badge bg-secondary">一般使用者</span>'}
```

修改後：
```javascript
${(() => {
  const roleName = rolesMap[user.role] || user.role;
  const badgeClass = user.role === 'admin' ? 'bg-danger' : 
                    user.role === 'boss' ? 'bg-primary' :
                    user.role === 'salesperson' ? 'bg-info' :
                    user.role === 'user' ? 'bg-secondary' : 'bg-dark';
  return '<span class="badge ' + badgeClass + '">' + roleName + '</span>';
})()}
```

## 功能特點

### 1. 動態角色選單
- ✅ 自動從 `roles` 表讀取所有啟用的角色
- ✅ 按照 `display_order` 和 `role_name` 排序
- ✅ 只顯示啟用的角色（`is_active = 1`）

### 2. 向後兼容
- ✅ 如果 `roles` 表不存在，自動回退到預設角色
- ✅ 不會因為資料庫結構問題導致頁面錯誤

### 3. 即時更新
- ✅ 在角色管理中新增角色後，立即可在使用者管理中選取
- ✅ 停用角色後，不會出現在選單中（但不影響已分配該角色的使用者）

## 常見問題

### Q: 為什麼角色選單還是只顯示 4 個預設角色？

**A: 可能的原因：**

1. **快取問題** - 清除瀏覽器快取並重新整理
2. **服務未重啟** - 確保已重啟服務
3. **檔案未更新** - 確認伺服器上的檔案是否為最新版本

**檢查方法：**
```bash
# 檢查服務狀態
sudo systemctl status project-system.service

# 查看日誌
sudo journalctl -u project-system.service -n 50
```

### Q: 停用的角色還會出現在選單中嗎？

**A: 不會。**
- 新增/編輯使用者時，只會顯示 `is_active = 1` 的角色
- 但已分配停用角色的使用者不受影響

### Q: 如何更改自訂角色的顯示順序？

**A: 在「角色管理」中編輯角色，修改「顯示順序」欄位。**
- 數字越小越靠前
- 相同順序按角色名稱排序

### Q: 自訂角色的徽章顏色如何設定？

**A: 目前自訂角色的徽章顏色預設為深灰色（bg-dark）。**

如需自訂顏色，可以：
1. 在 `roles` 表新增 `badge_color` 欄位
2. 修改 `users/index.ejs` 讀取該欄位
3. 或在角色管理功能中新增顏色選擇器

## 技術細節

### 資料流程

```
1. 使用者訪問 /users 或 /users/new
   ↓
2. 路由查詢 roles 表（SELECT ... WHERE is_active = 1）
   ↓
3. 將角色列表傳遞給視圖
   ↓
4. 視圖動態生成 <option> 元素
   ↓
5. 使用者選擇角色並提交表單
   ↓
6. 儲存 role_key 到 users.role 欄位
```

### SQL 查詢

**查詢角色列表：**
```sql
SELECT role_key, role_name 
FROM roles 
WHERE is_active = 1 
ORDER BY display_order ASC, role_name ASC
```

**查詢角色名稱映射：**
```sql
SELECT role_key, role_name 
FROM roles
```

### 錯誤處理

所有角色查詢都包含 try-catch 錯誤處理：
- 如果 `roles` 表不存在 → 使用預設角色
- 如果查詢失敗 → 記錄警告並使用預設角色
- 確保系統在任何情況下都能正常運作
