# 修復腳本執行問題說明

## 問題
當在 Linux 伺服器上執行腳本時出現錯誤：
```bash
-bash: ./fix-roles-table.sh: cannot execute: required file not found
```

## 原因
這個錯誤通常是因為檔案使用了 **Windows 行尾符號（CRLF）**，而不是 Linux 的 **Unix 行尾符號（LF）**。Linux 系統無法正確解析帶有 CRLF 的腳本檔案。

## 解決方法

### 方法 1：使用 dos2unix 工具（推薦）

如果伺服器已安裝 `dos2unix`：
```bash
dos2unix fix-roles-table.sh
chmod +x fix-roles-table.sh
./fix-roles-table.sh
```

如果未安裝，可以先安裝：
```bash
# Ubuntu/Debian
apt-get update && apt-get install -y dos2unix

# CentOS/RHEL
yum install -y dos2unix
```

### 方法 2：使用 sed 命令

```bash
sed -i 's/\r$//' fix-roles-table.sh
chmod +x fix-roles-table.sh
./fix-roles-table.sh
```

### 方法 3：使用 tr 命令

```bash
tr -d '\r' < fix-roles-table.sh > fix-roles-table-fixed.sh
chmod +x fix-roles-table-fixed.sh
./fix-roles-table-fixed.sh
```

### 方法 4：使用 vim 編輯器

```bash
vim fix-roles-table.sh
# 在 vim 中執行：
:set ff=unix
:wq

chmod +x fix-roles-table.sh
./fix-roles-table.sh
```

### 方法 5：使用 Perl（如果可用）

```bash
perl -pi -e 's/\r\n/\n/g' fix-roles-table.sh
chmod +x fix-roles-table.sh
./fix-roles-table.sh
```

## 一次性修復所有腳本

如果需要修復多個腳本檔案，可以使用：

```bash
# 使用 dos2unix
dos2unix *.sh

# 或使用 sed
find . -name "*.sh" -exec sed -i 's/\r$//' {} \;

# 然後添加執行權限
chmod +x *.sh
```

## 檢查檔案格式

可以使用以下命令檢查檔案的換行符號類型：

```bash
# 檢查檔案類型
file fix-roles-table.sh

# 或使用 od 命令查看
od -c fix-roles-table.sh | head

# 或使用 cat -A（會顯示行尾符號，$ 表示 LF，^M$ 表示 CRLF）
cat -A fix-roles-table.sh | head -5
```

## 預防措施

### 在 Git 中設定自動轉換

在專案根目錄的 `.gitattributes` 檔案中添加：
```
*.sh text eol=lf
```

這樣可以確保所有 `.sh` 檔案在提交到 Git 時使用 LF 行尾符號。

### 在編輯器中設定

如果使用 VS Code 或 Cursor：
1. 開啟設定
2. 搜尋 "EOL" 或 "行尾序列"
3. 設定為 "LF" 或 "Unix"

## 驗證修復

修復後，可以使用以下命令驗證：
```bash
# 檢查檔案類型，應該顯示 "ASCII text" 而不是 "ASCII text, with CRLF line terminators"
file fix-roles-table.sh

# 檢查 shebang 行是否正確
head -1 fix-roles-table.sh | od -c

# 應該看到：
# 0000000   #   !   /   b   i   n   /   b   a   s   h  \n
#                                           ^^^^
#                                       這是 LF（\n），不是 CRLF（\r\n）
```
