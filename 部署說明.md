# 快速部署指南

本文件說明如何從 Windows 開發環境快速部署到 Ubuntu 伺服器。

## 📋 前置準備

### 1. 修改伺服器資訊

在執行部署前，請先編輯 `快速部署到伺服器.ps1`，修改以下設定：

```powershell
$SERVER_USER = "root"              # 伺服器登入帳號
$SERVER_IP = "pbg-dev.m7r.tw"      # 伺服器 IP 或域名（請修改這裡！）
$SERVER_PATH = "/root/pbg-ins"     # 伺服器上的專案目錄
```

### 2. 安裝必要工具

**選項 A：使用 Git for Windows（推薦）**
- 下載並安裝 [Git for Windows](https://git-scm.com/download/win)
- 安裝時選擇「Use Git and optional Unix tools from the Command Prompt」
- 這會自動安裝 `ssh` 和 `rsync`

**選項 B：使用 WSL（Windows Subsystem for Linux）**
```powershell
# 在 PowerShell（管理員模式）執行
wsl --install
```

**選項 C：使用 Cygwin**
- 下載 [Cygwin](https://www.cygwin.com/)
- 安裝時選擇 `rsync` 和 `openssh` 套件

### 3. 設定 SSH 連線

**測試 SSH 連線：**
```powershell
ssh root@pbg-dev.m7r.tw
```

如果無法連線，請檢查：
- ✅ 伺服器 IP/域名正確
- ✅ SSH 服務運行中
- ✅ 防火牆允許 SSH (port 22)

**（選用）設定 SSH Key 免密碼登入：**
```powershell
# 生成 SSH Key（如果還沒有）
ssh-keygen -t rsa -b 4096

# 複製公鑰到伺服器
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh root@pbg-dev.m7r.tw "cat >> ~/.ssh/authorized_keys"
```

## 🚀 部署方式

### 方式一：使用 PowerShell 腳本（推薦）⭐

1. **開啟 PowerShell（以系統管理員身分執行）**
2. **允許執行腳本（首次使用時需要）：**
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

3. **執行部署腳本：**
   ```powershell
   .\快速部署到伺服器.ps1
   ```

**腳本會自動：**
- ✅ 測試 SSH 連線
- ✅ 複製檔案到伺服器（排除 node_modules、資料庫等）
- ✅ 在伺服器上執行 `deploy.sh`
- ✅ 驗證服務狀態

### 方式二：使用批次檔（.bat）

1. **編輯 `快速部署到伺服器.bat`**，修改伺服器資訊
2. **雙擊執行** `快速部署到伺服器.bat`

### 方式三：手動部署

```powershell
# 1. 複製文件到伺服器
rsync -av --exclude="node_modules" --exclude=".git" --exclude="*.log" `
  --exclude="data/*.db" --exclude="uploads/*" `
  . root@pbg-dev.m7r.tw:/root/pbg-ins/

# 2. SSH 登入伺服器
ssh root@pbg-dev.m7r.tw

# 3. 執行部署腳本
cd /root/pbg-ins
sudo ./deploy.sh
```

## ❌ 常見問題

### 問題 1: `rsync: command not found`

**解決方式：**
- 安裝 Git for Windows 並確保選擇「Use Git and optional Unix tools」
- 或使用 `scp` 替代（速度較慢）：
  ```powershell
  scp -r * root@pbg-dev.m7r.tw:/root/pbg-ins/
  ```

### 問題 2: SSH 連線失敗

**解決方式：**
1. 檢查伺服器 IP/域名是否正確
2. 檢查防火牆設定
3. 嘗試手動 SSH 連線：
   ```powershell
   ssh -v root@pbg-dev.m7r.tw
   ```
4. 如果提示「Host key verification failed」：
   ```powershell
   ssh-keygen -R pbg-dev.m7r.tw
   ```

### 問題 3: PowerShell 腳本無法執行

**解決方式：**
```powershell
# 以管理員模式開啟 PowerShell，執行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 問題 4: 部署後服務無法啟動

**解決方式：**
```bash
# SSH 登入伺服器後執行
sudo journalctl -u invoice-bonus-system.service -n 100 --no-pager
```

查看錯誤訊息，常見問題：
- **資料庫錯誤**：檢查 `/opt/invoice-bonus-system/data/` 權限
- **端口被佔用**：檢查是否有其他服務使用 port 3000
  ```bash
  sudo lsof -i:3000
  ```
- **缺少依賴**：重新安裝依賴
  ```bash
  cd /opt/invoice-bonus-system
  npm install
  ```

### 問題 5: 資料庫欄位缺失

**解決方式：**

`deploy.sh` 已經會自動處理，但如果仍有問題：

```bash
# SSH 登入伺服器後執行
cd /opt/invoice-bonus-system

# 檢查欄位是否存在
sqlite3 data/invoice_bonus.db "PRAGMA table_info(projects);" | grep expected_invoice_year_month

# 如果沒有，重新執行部署
sudo ./deploy.sh
```

## 🎯 最佳實踐

1. **定期備份**
   - 部署前先執行備份：`sudo /opt/invoice-bonus-system/backup.sh`
   - 設定自動備份：`sudo /opt/invoice-bonus-system/setup-backup-timer.sh`

2. **測試環境**
   - 建議先在測試環境部署，確認無誤後再部署到正式環境

3. **版本控制**
   - 使用 Git 管理程式碼
   - 每次部署前建立 Git tag：
     ```powershell
     git tag -a v1.7.1 -m "版本 1.7.1 - 部署流程優化"
     git push origin v1.7.1
     ```

4. **監控**
   - 定期檢查服務狀態
   - 設定異常告警（可使用 systemd 的 OnFailure）
