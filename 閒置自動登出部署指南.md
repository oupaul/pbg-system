# 閒置自動登出功能 - 快速部署指南

## 📦 部署步驟

### 1. 上傳檔案到伺服器

```bash
# 從開發環境上傳到伺服器
scp -r ./* username@your-server:/path/to/project/
```

**需要上傳的檔案**：
- ✅ `migrations/migrate_system_settings.js`（已更新）
- ✅ `public/js/main.js`（已更新）
- ✅ `src/app.js`（已更新）
- ✅ `src/views/layout.ejs`（已更新）
- ✅ `src/views/settings/index.ejs`（已更新）
- ✅ `src/routes/settings.js`（已更新）
- ✅ `閒置自動登出功能說明.md`（新增）
- ✅ `README.md`（已更新）

### 2. 執行一鍵部署

```bash
# 登入伺服器
ssh username@your-server

# 進入專案目錄
cd /path/to/project

# 執行一鍵部署
sudo ./deploy.sh
```

**部署腳本會自動執行**：
- 停止服務
- 更新依賴套件
- 執行資料庫遷移（自動添加閒置設定）
- 重啟服務
- 驗證服務狀態

### 3. 驗證部署

#### 檢查資料庫設定

```bash
cd /opt/invoice-bonus-system  # 或您的安裝目錄
sqlite3 data/database.sqlite "SELECT * FROM system_settings WHERE setting_key LIKE 'idle_%';"
```

**預期輸出**：
```
1|idle_timeout_minutes|30|number|使用者閒置自動登出時間（分鐘），0 表示停用閒置登出功能|2026-01-12 ...|2026-01-12 ...
2|idle_warning_minutes|2|number|閒置登出前的警告時間（分鐘），在達到閒置時間前多久顯示警告|2026-01-12 ...|2026-01-12 ...
```

#### 測試功能

1. **登入系統**
2. **打開瀏覽器開發者工具**（按 F12）
3. **查看控制台輸出**，應該看到：
   ```
   [閒置檢測] 已啟用
   [閒置檢測] 閒置時間: 30 分鐘
   [閒置檢測] 警告時間: 2 分鐘
   [閒置檢測] 自動登出時間: 30 分鐘後
   ```

4. **測試警告機制**（可選）：
   - 保持閒置 28 分鐘後，應該會看到警告對話框
   - 點擊「繼續使用」應該重置計時器
   - 或點擊「立即登出」應該立即登出

## ⚙️ 系統設定

### 調整閒置時間

1. 使用**管理員帳號**登入
2. 點擊左側選單「系統設定」
3. 找到「閒置自動登出設定」區塊
4. 調整「閒置時間」和「警告提前時間」
5. 點擊「儲存設定」
6. **通知所有使用者重新整理頁面**

### 建議設定值

| 使用情境 | 閒置時間 | 說明 |
|---------|---------|------|
| 標準辦公環境 | 30 分鐘 | 平衡安全性和使用體驗（推薦） |
| 高安全性環境 | 15 分鐘 | 更嚴格的安全控制 |
| 寬鬆環境 | 60 分鐘 | 較長的使用時間 |
| 停用功能 | 0 分鐘 | 完全停用閒置登出 |

### 停用功能

如果不需要此功能：

1. 進入「系統設定」頁面
2. 將「閒置時間（分鐘）」設定為 **0**
3. 點擊「儲存設定」
4. 通知使用者重新整理頁面

## 🔍 疑難排解

### 問題 1：控制台沒有顯示閒置檢測訊息

**可能原因**：
- 未登入系統
- JavaScript 檔案未更新
- 瀏覽器快取問題

**解決方案**：
```bash
# 1. 確認 main.js 已更新
ls -l /opt/invoice-bonus-system/public/js/main.js

# 2. 重啟服務
sudo systemctl restart invoice-bonus-system.service

# 3. 清除瀏覽器快取（Ctrl+Shift+R 或 Cmd+Shift+R）
```

### 問題 2：系統設定頁面沒有顯示閒置設定

**可能原因**：
- 資料庫遷移未執行
- 視圖檔案未更新

**解決方案**：
```bash
# 手動執行遷移
cd /opt/invoice-bonus-system
npm run migrate:system-settings

# 重啟服務
sudo systemctl restart invoice-bonus-system.service
```

### 問題 3：設定變更後沒有生效

**可能原因**：
- 瀏覽器快取
- 未重新整理頁面

**解決方案**：
- 按 **Ctrl+Shift+R**（Windows/Linux）或 **Cmd+Shift+R**（Mac）強制重新整理
- 或清除瀏覽器快取後重新載入頁面

### 問題 4：警告對話框無法關閉

**說明**：這是正常行為！警告對話框設計為無法通過點擊外部或按 ESC 關閉，必須選擇：
- 「繼續使用」- 延長會話
- 「立即登出」- 主動登出
