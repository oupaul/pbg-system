# 儀表板檢視模式與獨立加總功能說明

**更新日期**：2026-02-12

本文件說明儀表板檢視模式、業務獨立加總、專案類型分布控制，以及相關權限與顯示邏輯的實作。

---

## 一、儀表板檢視模式（`roles.dashboard_view_mode`）

依角色設定決定儀表板顯示方式，可動態新增角色，不寫死角色名稱。

| 模式 | 說明 | 適用角色範例 |
|------|------|--------------|
| `all_and_separate` | 全部專案 + 獨立業務區塊 | boss, user, admin |
| `exclude_separate` | 僅非獨立專案（主區塊排除獨立業務） | pm_manager |
| `none` | 不顯示儀表板，顯示歡迎頁 | salesperson |

### 主區塊排除邏輯

- **`exclude_separate`** 或 **`all_and_separate`** 且存在獨立業務時，主區塊（七個方塊、專案類型分布、最近專案等）**排除**獨立業務專案，避免與獨立區塊重複計算。
- 獨立業務區塊僅在 `all_and_separate` 模式下顯示。

---

## 二、業務獨立加總（`salespeople.show_separate_dashboard`）

- 勾選「儀表板獨立加總」的業務員，其專案在 `all_and_separate` 模式下會顯示於獨立區塊。
- 獨立區塊顯示：專案數（可點擊連結至專案管理篩選）、總金額、已開立發票、未開立發票、已收款、未收款。
- 專案數點擊後導向 `/projects?salesperson=業務名&year=年度`，自動篩選該業務專案。

---

## 三、權限與可見性

### 專案管理

- **獨立計算業務員的專案**：僅 admin、user、boss 可見；其他角色（如 pm_manager、salesperson）不可見。
- **業務下拉選單**：
  - 業務員：只顯示自己
  - 非 admin/user/boss：排除 `show_separate_dashboard = 1` 的業務
  - 離職人員：名稱後加 `(離職)` 備註
- **統計總計**：所有登入者皆顯示。

### 業務績效頁面

- 僅 **admin、user、boss** 可存取。
- 其他角色存取時顯示 403 無權限頁面。

### 專案毛利分析

- **獨立計算業務員的專案**：僅 admin、user、boss 可見於專案明細、業務彙總、類型彙總、報表群組彙總。
- 其他角色不顯示獨立計算業務的專案。

---

## 四、專案類型分布（儀表板）

- 新增 `project_types.show_in_dashboard` 欄位（1=顯示，0=不顯示）。
- 儀表板「專案類型分布」僅顯示 `show_in_dashboard = 1` 的類型。
- 於**類型管理**頁面可勾選/取消「儀表板專案類型分布顯示」。

### Migration

```bash
npm run migrate:project-types-show-in-dashboard
```

---

## 五、應收帳款與收款提醒

- **應收帳款帳齡分析**、**收款提醒**：始終包含**全部**發票（含獨立加總業務），以掌握完整應收狀況。
- 不受 `exclude_separate` 或 `all_and_separate` 影響。

---

## 六、儀表板卡片顏色

| 項目 | 顏色 |
|------|------|
| 未開立發票金額 | 橘色漸層（#fd7e14 → #e8590c） |
| 已開立發票未收款總額 | 紅色（bg-danger） |
| 待發放獎金 | 黃色（bg-warning） |
