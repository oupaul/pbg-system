# 發票作廢與折讓功能說明

## 更新日期
2026-02-07

## 功能概述

支援發票作廢、作廢重開、整筆折讓，保留完整軌跡，已開立總額與應收僅計算有效發票。

## 功能特點

### 作廢
- 將發票標記為「作廢」，不計入已開立總額與應收款項
- 可填寫作廢日期與原因
- 自動清除預計收款日

### 作廢重開
- 原發票作廢後建立新發票，保留兩者關聯
- 新發票可帶入原發票部分資料（日期、金額等可修改）
- 發票列表顯示「→ 重開 發票號碼」連結
- 自動清除原發票預計收款日

### 整筆折讓
- 將發票標記為「整筆折讓」，保留原金額供查詢但不計入總額
- 可填寫折讓日期與原因
- 自動清除預計收款日

## 操作位置

專案詳情頁 → 發票明細 → 有效發票的「更多」下拉選單：
- 作廢
- 作廢重開
- 整筆折讓

## 資料庫

### 新增欄位（invoices 表）
| 欄位 | 類型 | 說明 |
|------|------|------|
| status | TEXT | 有效、作廢、整筆折讓 |
| voided_at | TEXT | 作廢/折讓日期 |
| void_reason | TEXT | 作廢/折讓原因 |
| replacement_invoice_id | INTEGER | 作廢重開時的新發票 ID |
| original_invoice_id | INTEGER | 作廢重開時的原發票 ID |

### 遷移
- **腳本**：`migrations/migrate_invoice_status.js`
- **指令**：`npm run migrate:invoice-status` 或 `node migrations/migrate_invoice_status.js`
- **部署**：deploy.sh 已整合，更新部署時自動執行

## 計算邏輯

- **已開立發票總額**：僅計算 status = 有效 的發票
- **應收款項**：已開立 − 已收款 − 銷貨折讓（已開立亦僅計有效發票）
- 作廢/整筆折讓的發票仍保留於列表（灰色顯示），供稽核查詢
