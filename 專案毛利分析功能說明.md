# 專案毛利分析功能說明

## 更新日期
2026-02-13

---

## 一、功能概述

專案毛利分析依「方案 A」計算，以專案未稅價格作為收入，扣除專案成本後得出毛利與毛利率，便於檢視各專案、業務、類型的獲利狀況。支援毛利警示閾值、Excel 及 PDF 匯出、專案明細排序功能。

---

## 二、計算公式

| 項目 | 說明 |
|------|------|
| 收入 | 專案未稅價格（`price_without_tax`） |
| 成本 | 專案成本總和（`costs` 表 `amount` 加總） |
| 毛利 | 收入 − 成本 |
| 毛利率 | 毛利 ÷ 收入 × 100% |

---

## 三、顯示位置

導覽列 → **毛利分析**（所有已登入使用者可見，資料依角色權限過濾）

---

## 四、報表內容

### 彙總卡片

- 總收入（未稅）
- 總成本
- 總毛利
- 平均毛利率

### 專案明細分頁

| 欄位 | 說明 |
|------|------|
| 專案編號 | 可點擊連結至專案詳情 |
| 客戶 | 專案所屬客戶（公司名稱） |
| 專案名稱 | 專案說明 |
| 類型 | 專案類型（標籤顏色與專案管理一致，依專案類型管理設定） |
| 毛利率 | 百分比，低於該專案所屬類型閾值時顯示警示 |
| 毛利 | 收入 − 成本 |
| 收入（未稅） | 專案未稅價格 |
| 成本 | 專案成本總和 |
| 簽約年度 | 專案簽約年度 |
| 狀態 | 專案狀態：已結案（灰）、未結案（黃）、取消（紅） |

**排序功能**：點擊欄位標題可依該欄排序（升序/降序），預設依毛利由高到低排序。排序圖示：↕（未選）、↑（升序）、↓（降序）。

### 依業務彙總分頁

依業務人員彙總專案數、總收入、總成本、總毛利、毛利率，可連結至該業務的專案列表。

### 依類型彙總分頁

依專案類型彙總專案數、總收入、總成本、總毛利、毛利率。

### 依群組彙總分頁

依**報表群組**彙總專案數、總收入、總成本、總毛利、毛利率。同一群組的專案會合併計算；未歸屬任何群組的專案列在「未分群」。可於 **報表群組管理** 建立群組，並在專案新增/編輯時選擇「報表群組」。

---

## 五、毛利警示閾值

- 可於 **專案類型管理** 為各類型設定「毛利警示閾值(%)」
- 警示針對**每個專案**：若該專案毛利率低於其所屬類型的閾值，於專案明細分頁顯示警示（黃底、紅字、警示 badge）
- 閾值不是針對類型彙總，而是針對單一專案與其類型對應的閾值比較

---

## 六、Excel 匯出

- 路徑：`GET /gross-profit/export?year=2024` 或 `?year=all`，可加 `&status=未結案` 或 `&status=已結案`
- 檔名：`毛利分析_2024.xlsx`、`毛利分析_全部.xlsx`，或帶狀態如 `毛利分析_2024_未結案.xlsx`
- 四個工作表：**專案明細**、**依業務彙總**、**依類型彙總**、**依群組彙總**
- 依目前頁面篩選的年度與狀態匯出

---

## 七、PDF 匯出

- 路徑：`GET /gross-profit/export/pdf?year=2024` 或 `?year=all`，可加 `&status=未結案` 或 `&status=已結案`
- 檔名：`毛利分析_2024.pdf`、`毛利分析_全部.pdf`，或帶狀態如 `毛利分析_2024_已結案.pdf`
- 內容（由上而下）：標題、彙總資訊（總收入、總成本、總毛利、平均毛利率）、**依群組彙總**表格、**專案明細**表格（橫向 A4）
- 依群組彙總排在專案明細上方；專案明細列高 40pt 以容納專案名稱換行，數值欄位寬度已優化以完整顯示
- **版面**：表格置中於 A4 頁面；每頁有標題與頁碼（第 N 頁）；換頁時表格標題列重複顯示；標題欄位文字水平與垂直置中對齊
- 依目前頁面篩選的年度與狀態匯出

---

## 八、年度篩選

- **全部年度**：不限定專案簽約年度
- **指定年度**：僅統計該年度專案

---

## 八之一、狀態篩選

- **全部**：不依專案狀態篩選（含未結案、已結案、取消等）
- **未結案**：僅顯示狀態為「未結案」的專案（含 `status` 為空視為未結案）
- **已結案**：僅顯示狀態為「已結案」的專案

與年度篩選可同時使用；Excel／PDF 匯出會依目前頁面所選的年度與狀態一併匯出，檔名會帶入狀態（例如 `毛利分析_2024_未結案.xlsx`）。

---

## 八之二、重設按鈕

- 篩選列提供「重設」按鈕，點擊後清空年度與狀態篩選，回到「全部年度」「全部」狀態

---

## 八之三、手機版與響應式

- 篩選與按鈕區塊在手機版會自動換行、不超出版面
- Excel、PDF 按鈕在小螢幕（<768px）僅顯示圖示以節省空間，滑過可看完整說明
- 相關檔案：`src/views/gross-profit/index.ejs`、`public/css/style.css`（`.gross-profit-toolbar`）

---

## 九、權限

所有已登入使用者均可檢視毛利分析頁面，但資料會依角色權限過濾：

- **salesperson（業務員）**：僅能看到自己負責的專案毛利（`salesperson_id` 匹配）
- **boss（老闆）**：可看到所有專案毛利
- **admin（管理員）**：可看到所有專案毛利
- **user（一般使用者）**：可看到所有專案毛利

Excel 與 PDF 匯出也會套用相同的權限過濾。

- `src/routes/reportGroups.js` - 報表群組管理（僅 admin）
- `src/views/gross-profit/index.ejs` - 視圖
- `src/views/reportGroups/index.ejs` - 報表群組管理視圖
- `src/app.js` - 路由註冊
- `src/views/layout.ejs` - 導覽連結
- `migrations/migrate_report_groups.js` - 報表群組與專案欄位遷移（首次使用依群組彙總前請執行：`node migrations/migrate_report_groups.js`）

---

## 十一、資料來源

- **收入**：`projects.price_without_tax`
- **成本**：`costs` 表，每筆專案可有多筆成本記錄，由專案詳情頁「新增成本」維護
- **客戶**：`customers` 表，透過 `projects.customer_id` 取得 `company_name`
