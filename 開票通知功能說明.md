# 開票通知功能說明與測試

## 📋 功能概述

系統會在儀表板（首頁）自動顯示**即將到期的預計開票提醒**，幫助業務人員及時處理發票開立。

---

## ⚙️ 通知邏輯

### 觸發條件（三個條件都需滿足）

1. **時間條件**：當月剩餘天數 ≤ 6 天（即倒數第 7 天開始）
2. **專案條件**：專案的「預計開票年月」= 當前月份
3. **狀態條件**：專案狀態 = 「未結案」

### 通知時間範圍

**v1.7.6 更新：** 通知已從倒數第 3 天提前到倒數第 7 天

| 月份剩餘天數 | 是否顯示通知 | 說明 |
|-------------|-------------|------|
| 7 天或以上 | ❌ 否 | 尚未進入通知期 |
| 6 天 | ✅ 是 | **開始顯示**（倒數第 7 天） |
| 5 天 | ✅ 是 | |
| 4 天 | ✅ 是 | |
| 3 天 | ✅ 是 | |
| 2 天 | ✅ 是 | |
| 1 天 | ✅ 是 | |
| 0 天（當天） | ✅ 是 | 月底最後一天 |

### 範例計算

**假設今天是 2025-12-26（12月有31天）：**
- 剩餘天數 = 31 - 26 = 5 天
- 條件：5 ≤ 6 ✅
- **結果：會顯示通知**

**假設今天是 2025-12-24（12月有31天）：**
- 剩餘天數 = 31 - 24 = 7 天
- 條件：7 ≤ 6 ❌
- **結果：不會顯示通知**

## 🧪 測試步驟

### 步驟 1：重新部署最新代碼

```bash
# SSH 登入伺服器
ssh root@your-server

# 上傳最新檔案
cd /opt/invoice-bonus-system

# 重啟服務
sudo systemctl restart invoice-bonus-system

# 驗證服務狀態
sudo systemctl status invoice-bonus-system
```

### 步驟 2：設定測試專案

1. **登入系統**並進入**專案管理**
2. **選擇一個未結案的專案**（或新增測試專案）
3. **點擊專案**進入詳細頁面
4. 在**「業務預計開立發票年月」**區塊：
   - 選擇當前月份（例如：2025-12）
   - 點擊「更新」按鈕
5. **確認**專案狀態為「未結案」

### 步驟 3：查看儀表板

1. **返回首頁（儀表板）**
2. **查看調試資訊區塊**（藍色資訊框）：
   ```
   通知系統狀態（調試資訊）
   ✓ 當前月份：2025-12
   ✓ 當月剩餘天數：5 天
   ✓ 通知條件：剩餘 ≤ 6 天（即倒數第 7 天開始）
   ✓ 是否顯示通知：✓ 是
   ✓ 符合條件的專案數：1 個
   ```

3. **如果條件滿足**，應該會看到**黃色警告框**：
   ```
   🔔 開票提醒通知
   本月還剩 X 天！
   以下 N 個專案預計在 2025-12 開立發票，請儘速處理：
   [專案列表表格]
   ```

## ❓ 常見問題

### Q1: 我設定了預計開票年月，但沒有顯示通知？

**檢查清單：**
- [ ] 是否在月底倒數 7 天內？（剩餘 ≤ 6 天）
- [ ] 專案的「預計開票年月」是否為當前月份？
- [ ] 專案狀態是否為「未結案」？
- [ ] 是否重新整理了頁面？
- [ ] 是否重啟了服務？

**調試方法：**
查看藍色調試資訊框，檢查各項條件的狀態。

### Q2: 調試資訊顯示「符合條件的專案數：0 個」？

**可能原因：**
1. **沒有專案設定預計開票年月**
   - 解決：進入專案詳細頁面，設定「預計開票年月」

2. **預計開票年月不是當月**
   - 解決：將預計開票年月改為當前月份（如：2025-12）

3. **專案已結案**
   - 解決：確認專案狀態為「未結案」

4. **資料庫未更新**
   - 解決：重啟服務 `sudo systemctl restart invoice-bonus-system`

### Q3: 如何調整通知提前天數？

**臨時調整（需要修改代碼）：**

編輯 `/opt/invoice-bonus-system/src/routes/index.js`：

```javascript
// 第 120 行附近
const notificationDaysBeforeMonth = 6; // 改為你想要的值

// 範例：
// 6 = 倒數第 7 天開始（預設）
// 9 = 倒數第 10 天開始
// 2 = 倒數第 3 天開始（原始設定）
```

修改後重啟服務：
```bash
sudo systemctl restart invoice-bonus-system
```

**永久方案（未來功能）：**
我們計劃在後續版本添加「系統設定」功能，允許在網頁介面直接設定通知天數，無需修改代碼。

### Q4: 通知會發送郵件嗎？

**目前版本：** 
- ❌ 不會發送郵件
- ✅ 只在儀表板顯示

**未來計劃：**
- 郵件通知
- LINE 通知
- 推播通知

### Q5: 業務員和老闆角色看得到通知嗎？

**不會。**

業務員（salesperson）和老闆（boss）角色無法看到儀表板（首頁），因此看不到開票提醒通知。

如果需要讓這些角色也看到提醒，請提出需求，我們可以在專案管理頁面也顯示類似的提醒。

## 📊 資料查詢邏輯

### SQL 查詢

```sql
SELECT 
  id,
  project_code,
  project_name,
  project_type,
  salesperson_name,
  price_with_tax,
  expected_invoice_year_month,
  status,
  total_invoiced,
  (price_with_tax - total_invoiced) as uninvoiced_amount
FROM v_project_summary
WHERE expected_invoice_year_month = '2025-12'  -- 當前月份
  AND status = '未結案'                        -- 未結案
ORDER BY price_with_tax DESC                   -- 金額大的優先
```

### 手動驗證

可以使用以下命令檢查是否有符合條件的專案：

```bash
# SSH 登入伺服器
ssh root@your-server

# 檢查資料庫
sqlite3 /opt/invoice-bonus-system/data/invoice_bonus.db <<EOF
SELECT 
  project_code, 
  project_name, 
  expected_invoice_year_month, 
  status,
  price_with_tax
FROM projects
WHERE expected_invoice_year_month = strftime('%Y-%m', 'now', 'localtime')
  AND status = '未結案';
EOF
```
