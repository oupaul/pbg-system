# 備份目錄動態偵測修復說明

## 問題描述

前端頁面的備份與還原功能無法正確顯示備份檔案，因為系統中硬編碼了備份目錄路徑 `/opt/invoice-bonus-backups`，但實際部署時使用的是 `/opt/project-system-backups` 或其他自訂名稱。

### 原始問題

在 `src/services/BackupRestoreService.js` 中：

```javascript
const INSTALL_DIR = '/opt/invoice-bonus-system';
const BACKUP_DIR = '/opt/invoice-bonus-backups';
```

在 `src/routes/backupRestore.js` 中：

```javascript
const BACKUP_DIR = '/opt/invoice-bonus-backups';
```

這些硬編碼的路徑導致：
- ❌ 備份檔案列表為空
- ❌ 無法下載備份檔案
- ❌ 無法還原備份
- ❌ 無法刪除備份

## 解決方案

### 1. BackupRestoreService.js - 動態偵測邏輯

**新增功能**：

#### detectInstallDir() - 偵測安裝目錄
```javascript
function detectInstallDir() {
  // 如果當前目錄在 /opt 下，就使用當前目錄
  if (PROJECT_DIR.startsWith('/opt/')) {
    return PROJECT_DIR;
  }
  
  // 否則嘗試在 /opt 下尋找可能的安裝目錄
  const possibleDirs = [
    '/opt/project-system',
    '/opt/invoice-bonus-system',
    '/opt/fund-weekly-report'
  ];
  
  for (const dir of possibleDirs) {
    if (fs.existsSync(dir) && fs.existsSync(path.join(dir, 'package.json'))) {
      return dir;
    }
  }
  
  return null;
}
```

**偵測邏輯**：
1. 優先使用當前運行目錄（如果在 `/opt` 下）
2. 嘗試已知的安裝目錄
3. 返回 `null`（表示開發環境）

#### detectBackupDir() - 偵測備份目錄
```javascript
function detectBackupDir(installDir) {
  if (!installDir) {
    // 如果沒有安裝目錄，使用專案目錄下的 backups
    return path.join(PROJECT_DIR, 'backups');
  }
  
  // 根據安裝目錄名稱生成備份目錄名稱
  const installDirName = path.basename(installDir);
  const backupDir = `/opt/${installDirName}-backups`;
  
  // 如果備份目錄存在，就使用它
  if (fs.existsSync(backupDir)) {
    return backupDir;
  }
  
  // 否則檢查其他可能的備份目錄
  const possibleBackupDirs = [
    `/opt/${installDirName}-backups`,
    '/opt/project-system-backups',
    '/opt/invoice-bonus-backups',
    '/opt/fund-weekly-backups'
  ];
  
  for (const dir of possibleBackupDirs) {
    if (fs.existsSync(dir)) {
      return dir;
    }
  }
  
  // 如果都不存在，返回預設的備份目錄路徑（可能需要創建）
  return backupDir;
}
```

**偵測邏輯**：
1. 開發環境：使用 `PROJECT_DIR/backups`
2. 生產環境：根據安裝目錄名稱生成對應的備份目錄
   - `/opt/project-system` → `/opt/project-system-backups`
   - `/opt/invoice-bonus-system` → `/opt/invoice-bonus-backups`
3. 檢查多個可能的備份目錄位置
4. 返回找到的第一個存在的目錄，或預設路徑

### 2. backupRestore.js - 使用服務邏輯

**修改前**：
```javascript
// 查找備份檔案
const BACKUP_DIR = '/opt/invoice-bonus-backups';
const PROJECT_DIR = process.cwd();

let backupPath = null;
// 優先檢查 /opt/invoice-bonus-backups
if (fs.existsSync(BACKUP_DIR)) {
  const testPath = path.join(BACKUP_DIR, filename);
  if (fs.existsSync(testPath)) {
    backupPath = testPath;
  }
}
// ... 更多檢查邏輯
```

**修改後**：
```javascript
// 查找備份檔案 - 使用 BackupRestoreService 的邏輯
const backups = BackupRestoreService.listBackups();
const backup = backups.find(b => b.filename === filename);

if (!backup || !backup.path) {
  return res.status(404).send('找不到備份檔案');
}

const backupPath = backup.path;
```

**優點**：
- 統一使用 BackupRestoreService 的邏輯
- 避免重複的偵測代碼
- 更容易維護

### 偵測邏輯流程圖

```
開始
  ↓
檢查當前目錄是否在 /opt 下？
  ├─ 是 → 使用當前目錄
  └─ 否 → 檢查已知安裝目錄列表
            ↓
          找到有效的安裝目錄？
            ├─ 是 → 使用找到的目錄
            └─ 否 → 返回 null（開發環境）
  ↓
根據安裝目錄生成備份目錄名稱
  ↓
檢查備份目錄是否存在？
  ├─ 是 → 使用該目錄
  └─ 否 → 檢查其他可能的備份目錄
            ↓
          找到存在的備份目錄？
            ├─ 是 → 使用找到的目錄
            └─ 否 → 使用預設路徑
  ↓

```

### 多重備份目錄支援

系統會按以下順序檢查備份目錄：

1. **主要備份目錄**（根據安裝目錄名稱）
   - `/opt/project-system-backups`

2. **備選備份目錄**（已知的其他備份目錄）
   - `/opt/invoice-bonus-backups`
   - `/opt/fund-weekly-backups`

3. **開發環境備份目錄**
   - `PROJECT_DIR/backups`

使用找到的第一個存在的目錄。

確保系統在任何情況下都能正常運作。

## 常見問題

### Q1: 系統如何判斷使用哪個備份目錄？

**A**: 系統按以下優先順序：

1. 如果當前目錄在 `/opt` 下 → 使用 `當前目錄名稱-backups`
2. 檢查已知安裝目錄 → 使用 `安裝目錄名稱-backups`
3. 檢查所有可能的備份目錄 → 使用第一個存在的
4. 開發環境 → 使用 `PROJECT_DIR/backups`

### Q2: 如果有多個備份目錄會怎樣？

**A**: 系統會使用找到的第一個備份目錄，但 `listBackups()` 函數會檢查所有可能的目錄，並合併所有找到的備份檔案。

### Q3: 升級後需要手動配置嗎？

**A**: 不需要。系統會自動偵測正確的目錄，無需任何配置。

### Q4: 舊的備份檔案會受影響嗎？

**A**: 不會。所有舊備份都會被自動找到並顯示在列表中，無論它們在哪個備份目錄。

### Q5: 開發環境和生產環境有什麼不同？

**A**: 
- **開發環境**：使用 `PROJECT_DIR/backups`
- **生產環境**：使用 `/opt/<安裝目錄名稱>-backups`

系統自動判斷，無需手動設定。

### Q6: 如何新增自訂的備份目錄？

**A**: 在 `detectBackupDir()` 函數的 `possibleBackupDirs` 陣列中新增自訂路徑：

```javascript
const possibleBackupDirs = [
  `/opt/${installDirName}-backups`,
  '/opt/project-system-backups',
  '/opt/invoice-bonus-backups',
  '/opt/fund-weekly-backups',
  '/path/to/your/custom/backup/dir'  // 新增自訂路徑
];
```

### Q7: 備份目錄不存在會自動創建嗎？

**A**: 會。`listBackups()` 函數會嘗試創建備份目錄（需要適當權限）：

```javascript
try {
  fs.mkdirSync(BACKUP_DIR, { recursive: true, mode: 0o755 });
  console.log('[BackupRestoreService] 成功創建備份目錄');
} catch (err) {
  console.warn('[BackupRestoreService] 無法創建備份目錄:', err.message);
}
```

## 效能影響

### 目錄偵測成本

- **模組載入時執行一次**：只在服務啟動時執行
- **時間複雜度**：O(n)，n 為可能的目錄數量（通常 < 5）
- **效能影響**：可忽略（< 1ms）

### 備份列表查詢

- **每次查詢都重新掃描**：確保資料最新
- **掃描範圍**：所有可能的備份目錄
- **效能影響**：極小（通常 < 10ms）

## 後續改進建議

### 1. 快取備份列表

減少重複的檔案系統掃描：

```javascript
let backupListCache = null;
let cacheTimestamp = 0;
const CACHE_TTL = 5000; // 5秒

static listBackups() {
  const now = Date.now();
  if (backupListCache && (now - cacheTimestamp) < CACHE_TTL) {
    return backupListCache;
  }
  
  backupListCache = this._scanBackupDirs();
  cacheTimestamp = now;
  return backupListCache;
}
```
